Project Abstract

CyberSherlock is an AI-assisted External Attack Surface Management (EASM) system developed at the University of Bahrain that addresses organizations' struggle to identify unmanaged assets and evaluate their associated risks. It utilizes passive Open-Source Intelligence (OSINT) techniques and HTML parsing to map infrastructure components such as DNS records, SSL certificates, and IP addresses. Built on Python and Flask, the system integrates a specialized AI security advisor aligned with OWASP standards. Testing on environments like OWASP Juice Shop demonstrated the system's efficiency in identifying critical vulnerabilities and outdated protocols within seconds.

The Blind Spot Problem

Organizations continuously publish data across websites, social media, repositories, and professional networks. This large amount of publicly accessible information creates new cybersecurity concerns as attackers can gather and correlate exposed data from multiple online sources.

02

Unmanaged External Assets

Developers constantly add new websites, services, and applications while older assets may be overlooked. Digital boundaries have become dynamic due to cloud instances, forgotten subdomains, and third-party APIs — creating an invisible perimeter.

03

Manual Analysis Doesn't Scale

Information such as employee details, email structures, technologies, DNS records, and infrastructure-related data may unintentionally reveal valuable insights about an organization's environment to any attacker.

04

No Structured Risk Visibility

Without automated scoring and AI-guided recommendations, security teams cannot effectively prioritize which exposures pose the greatest risk or determine whether publicly available information may introduce cybersecurity risks.

How It Works

From HTML File to
Complete Intelligence Report

A structured six-stage pipeline that transforms a saved webpage HTML into a complete external attack surface analysis — all using passive techniques only.

01

Upload HTML Source

A locally saved copy of the organization's main webpage HTML file is provided as the primary data source. The Flask application verifies the file and reads its content into memory for processing.

→
02

Extract Hosts & Domains

BeautifulSoup parses the HTML and scans tags for attributes like href, src, action, data-src, and srcset. Hostnames are extracted, normalized, and deduplicated. The most-repeated registered domain is selected as the main domain.

→
03

Collect Passive OSINT

The Passive OSINT Data Collection Module queries public APIs and databases for DNS records, WHOIS information, SSL certificate details, reverse DNS, ASN details, and IP ownership information — all without direct interaction with the target.

04

Analyze External Exposure

Collected OSINT data is analyzed to identify publicly exposed assets, third-party dependencies, unresolved hosts, and potentially risky external services associated with the target organization.

→
05

Generate Risk Score

A severity-weighted scoring algorithm evaluates each finding — exposed hosts, third-party dependencies, missing information, and detected vulnerabilities — to produce a consolidated Baseline Risk Score from 0–100 with category breakdowns.

→
06

AI Recommendations

The CyberSherlock AI Security Advisor processes all findings and generates prioritized, context-aware remediation recommendations aligned with OWASP security practices to help users understand and mitigate identified risks.

Can you justify the current Risk Score? Tell me which factor contributed most to this level.
AI
Based on the EASM scan results provided, the overall Risk Score is 22/100 (Low). The main contributor is the "Third-party hosts detected" finding with a severity of Medium. This indicates one external host was found in the saved HTML, which may expand external exposure. The finding suggests dependencies on external resources that are outside direct organizational control.
👤
Provide a top-3 priority action plan for the IT team to reduce our external attack surface.
AI
[SECURITY ALERT] Based on the results, I recommend:

1. Vulnerability Remediation: Review identified third-party hosts (e.g., external domains found in HTML source) and assess whether they are necessary dependencies.

2. Network Segmentation: Isolate and restrict access to sensitive areas to minimize the impact of external dependency exposure.

3. Regular Security Audits: Schedule periodic passive OSINT monitoring to detect new threats and changes to your external attack surface in a timely manner.
👤
Given that "Third-party hosts" are the main risk factor, provide a technical recommendation to harden web assets and mitigate potential supply chain attacks.
AI
Technical Recommendation:

1. Inventory and Monitoring: Conduct a comprehensive inventory of all third-party libraries, CDNs, and external scripts referenced in your web assets.

2. Vulnerability Scanning: Perform regular passive scans against known CVE databases for all identified third-party dependencies to detect outdated or vulnerable components.

3. Subresource Integrity (SRI): Implement SRI hashes on all externally loaded scripts and stylesheets to prevent tampering — as recommended by OWASP A06: Vulnerable and Outdated Components.
